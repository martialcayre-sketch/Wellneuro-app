# 02 — Objectif cible issu du brainstorming

## Vision condensée

WellNeuro 3.0 doit être un **système longitudinal d’aide à la décision neuronutritionnelle**, non diagnostique, validé par le praticien.

Question produit centrale :

```text
Quelle est la prochaine action utile, réaliste, sûre et compréhensible pour ce patient, dans les 21 prochains jours ?
```

## Promesse praticien

```text
Du questionnaire au protocole validé en 10 minutes.
```

Cette promesse inclut :

- résumé clinique ;
- priorités ;
- données manquantes ;
- signaux convergents ;
- signaux discordants ;
- proposition 21 jours ;
- documents prêts ;
- validation praticien.

## Promesse patient

```text
Je comprends ce que je dois faire aujourd’hui, sans être submergé.
```

Le patient ne doit pas être exposé à toute la complexité clinique. Il reçoit :

- une priorité actuelle ;
- une action principale ;
- un plan minimal ;
- une fiche utile ;
- un check-in court ;
- un langage rassurant.

## Philosophie produit

```text
Une app intelligente propose beaucoup.
Une app sage propose peu, au bon moment, avec justification.
```

## Piliers fonctionnels cibles

1. Fiche patient cockpit.
2. Profil fonctionnel consolidé non diagnostique.
3. Protocole adaptatif 21 jours.
4. Charge thérapeutique.
5. Données manquantes et biologie raisonnée.
6. Signaux convergents et discordants.
7. Documents multi-destinataires.
8. Compagnon patient minimal.
9. Momentum J7/J14/J21.
10. Modules avancés : clean label, Boussole alimentaire, messagerie contextualisée, copilotes IA.

## Vocabulaire à privilégier

- profil fonctionnel ;
- hypothèse fonctionnelle ;
- priorité actuelle ;
- protocole personnalisé ;
- proposition à valider ;
- exploration biologique à discuter ;
- données à documenter ;
- action prioritaire ;
- plan minimal ;
- ajustement J21.

## Vocabulaire à éviter

- diagnostic ;
- prescription automatique ;
- ordonnance automatique ;
- traitement prescrit par l’app ;
- risque élevé affiché au patient ;
- non-observance ;
- échec ;
- mauvais score ;
- protocole obligatoire.

## Métrique de réussite UX

Un praticien doit pouvoir ouvrir une fiche patient et répondre en moins de deux minutes :

1. Où en est ce patient ?
2. Quelle priorité traiter maintenant ?
3. Que faire pendant les 21 prochains jours ?
4. Qu’est-ce qui est trop chargé ?
5. Qu’est-ce qui manque pour décider ?
6. Quel document envoyer ?
