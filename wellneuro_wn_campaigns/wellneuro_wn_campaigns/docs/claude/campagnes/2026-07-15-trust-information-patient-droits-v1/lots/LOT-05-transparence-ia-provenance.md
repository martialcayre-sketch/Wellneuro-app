# LOT-05 — Transparence IA, validation humaine et provenance

## But

Rendre l’usage de l’IA et l’origine des contenus compréhensibles et exacts.

## Périmètre

- registre des cas d’usage ;
- badges ;
- modèle/prompt/version ;
- validation humaine ;
- explication patient ;
- contestation ;
- minimisation ;
- tests.

## Dépendances

- synthèse IA existante ;
- C1/C3 ;
- prompt caching ;
- contrats LOT-01.

## Étapes

1. Inventaire des appels IA.
2. Cartographie des données.
3. Définition des badges.
4. Blocage publication sans validation.
5. Vue « comprendre l’origine ».
6. Registre IA.
7. Tests patient-safe.
8. Revue fournisseur/contrat.

## Interdits

- badge faux ;
- « validé » sans trace ;
- publication autonome ;
- prompt secret exposé ;
- score de confiance patient ;
- information générique contredisant l’architecture réelle.
