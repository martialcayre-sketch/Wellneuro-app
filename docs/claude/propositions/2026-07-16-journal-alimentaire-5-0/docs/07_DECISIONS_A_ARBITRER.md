# Décisions à arbitrer avant campagne

## D1 — Nom patient

Recommandation : **Ma spirale alimentaire**.

Alternative plus sobre : **Mes repères alimentaires**.

## D2 — Politique par défaut

Recommandation : mode hybride 21 jours : panorama léger, focus action, consolidation.

## D3 — Définition de l’observation suffisante

Décider si les règles sont :

- globales ;
- spécifiques à chaque action ;
- spécifiques à chaque item SIIN ;
- ou combinées.

Recommandation : règles par `observationQuestionType`, versionnées.

## D4 — Photo

Recommandation :

- facultative ;
- traitement transitoire ;
- suppression de l’original après confirmation ;
- conservation d’une vignette uniquement sur choix explicite.

## D5 — Voix

Recommandation : suppression de l’audio, texte brut supprimé après extraction sauf choix explicite.

## D6 — Signatures entre épisodes

Recommandation : signatures durables liées à l’identité patient, mais modifiables et supprimables.

## D7 — Plan minimal

Recommandation : activation libre par le patient pour 1, 3 ou 7 jours ; le praticien voit uniquement le statut et la période, pas une justification obligatoire.

## D8 — Météo

Recommandation : praticien seulement, trois états et causes observables ; aucun score interne affichable.

## D9 — Notifications

Recommandation : notifications contextuelles limitées, paramétrables, désactivables et toujours accompagnées de « pourquoi maintenant ».

## D10 — Comparaison multi-épisodes

Recommandation : seulement instrument, mapping et politique compatibles ; sinon comparaison qualitative avec avertissement.

## D11 — Jumeau alimentaire

Recommandation : utiliser le nom **simulateur d’action** en production. Réserver « jumeau » au brainstorming pour éviter une promesse prédictive.

## D12 — Questionnaire J21

Décider quand répéter Q_ALI_01 et Q_ALI_02 :

- systématiquement ;
- selon la DecisionCard ;
- selon la question de phase.

Recommandation : assignation explicite dans le protocole, jamais automatique.
