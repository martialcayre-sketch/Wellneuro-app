# LOT-01 (alias JA-0T) — Validation terrain

> Id outillage : `LOT-01` (format requis par l'audit de campagne) ; le
> programme et le registre désignent ce lot sous l'alias **JA-0T**.

> Lot acté par l'adaptation contrepoint (A7-14) : cinq entretiens patients
> avant toute ligne de code du domaine, pour tester les hypothèses E1
> (boucle courte) et E5 (acceptabilité relationnelle) du doc 09, et la carte
> papier A6. La production de vision a dépassé la validation — ce lot
> rééquilibre.

## Règles de recueil (non négociables)

- **Aucun nom réel, aucune donnée patient dans le dépôt.** Les entretiens
  sont consignés sous des codes anonymes (P1…P5) avec âge approximatif et
  contexte générique uniquement (ex. « P2, ~50 ans, actif, consulte pour
  fatigue »).
- Consentement verbal explicite en début d'entretien : « J'aimerais vous
  poser quelques questions pour améliorer un futur outil de suivi ; rien
  n'est enregistré, aucune donnée de santé n'est notée. »
- L'entretien dure **10 minutes maximum**, en fin de consultation.
- Aucune promesse de fonctionnalité au patient.

## Trame d'entretien (10 minutes)

### Volet A — le vécu du suivi alimentaire (2 min, ouvre la discussion)

1. « Avez-vous déjà tenu un carnet ou une application de suivi alimentaire ?
   Qu'est-ce qui vous a fait arrêter — ou tenir ? »
2. « Quand je vous propose un changement alimentaire, qu'est-ce qui fait
   qu'il tient ou non dans votre quotidien ? »

### Volet B — la boucle courte E1 (4 min, montrer la carte A6)

Présenter la carte papier et une action exemple (« ajouter une source de
protéines au petit-déjeuner »). Puis :

3. « Avec ces trois cases — l'occasion s'est présentée ? c'était faisable ?
   qu'est-ce qui a compté ? — auriez-vous ce qu'il faut pour me raconter
   votre semaine ? Que manque-t-il ? »
4. « Combien de fois par semaine accepteriez-vous de remplir ça, au
   maximum ? » *(calibre le budget d'attention)*
5. « Si l'application vous disait certains jours : “rien à noter
   aujourd'hui, nous en savons assez”, qu'en penseriez-vous ? »
   *(droit au silence)*

### Volet C — l'acceptabilité E5 (3 min)

6. « Est-ce que ce suivi vous semblerait être une aide pour notre travail
   ensemble, ou plutôt une surveillance de vos repas ? Qu'est-ce qui ferait
   basculer d'un côté ou de l'autre ? »
7. « Si une semaine est difficile et que vous ne notez rien, qu'est-ce que
   vous aimeriez que je voie de mon côté — et que je ne voie pas ? »
8. « Après le point d'étape, aimeriez-vous savoir ce que vos notes ont
   changé à ma décision ? » *(retour de décision)*

### Clôture (1 min)

9. « Papier ou téléphone, vous préféreriez quoi pour ça ? »

## Relevé par entretien

| Élément | Réponse |
|---|---|
| Code (P1…P5) / date / contexte générique | |
| Carnet déjà tenu ? cause d'arrêt | |
| Ce qui fait tenir une action (frictions citées spontanément) | |
| Les 3 cases suffisent-elles ? manque cité | |
| Budget accepté (traces/semaine) | |
| Réaction au silence utile | |
| Aide vs surveillance — bascule citée | |
| Semaine difficile : voir / ne pas voir | |
| Intérêt pour le retour de décision | |
| Papier ou téléphone | |
| Verbatim marquant (anonyme) | |

## Critères de go/no-go du noyau

**GO** si, sur au moins 4 entretiens sur 5 :

- les trois cases sont jugées suffisantes ou presque (manques mineurs) ;
- le budget accepté est ≥ 3 traces/semaine ;
- « aide » l'emporte sur « surveillance » ;
- le retour de décision suscite de l'intérêt.

**NO-GO ou reformulation** si : la boucle courte est jugée inutile sans
description des repas ; le sentiment de surveillance domine ; le budget
accepté est < 2 traces/semaine. Dans ce cas, consigner ce qui devrait
changer avant de re-tester — **ne pas compiler JA5-01**.

Résultat consigné en fin de lot dans ce fichier (section « Enseignements »,
à ajouter), et reporté au registre si le cap doit bouger.
